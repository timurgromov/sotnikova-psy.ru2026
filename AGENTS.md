# AGENTS

Обязательные правила для любого нового агента, работающего в этом репозитории.

## Главный принцип

Если пользователь просит изменить сайт, задача считается завершённой только тогда, когда:

1. Изменены исходники.
2. Изменения проверены локально.
3. Исходная ветка закоммичена и запушена.
4. GitHub Pages обновлён и запушен.

Недопустимо останавливаться после локальных правок или после пуша только исходников.

## Ветки проекта

- Исходники сайта: `gh-pages-source`
- Публичный деплой GitHub Pages: `gh-pages`

## Обязательный workflow для правок сайта

1. Проверить текущую ветку и статус репозитория.
2. Внести изменения в исходники в ветке `gh-pages-source`.
3. Проверить результат локально, минимум через `npm run build`.
4. Сделать `git commit` в `gh-pages-source`.
5. Сделать `git push origin gh-pages-source`.
6. Собрать свежий билд сайта.
7. Обновить содержимое ветки `gh-pages` файлами из `dist/`.
8. Не удалять служебные файлы GitHub Pages, например `CNAME` и `.nojekyll`, если они уже есть в `gh-pages`.
9. Сделать `git commit` в `gh-pages`.
10. Сделать `git push origin gh-pages`.

## Что нельзя делать

- Нельзя считать задачу завершённой после одного только `commit` или одного только `push` исходников.
- Нельзя игнорировать деплой в `gh-pages`, если пользователь просил изменить сайт.
- Нельзя терять `CNAME`, `.nojekyll` и другие файлы, нужные для GitHub Pages.
- Нельзя описывать уже существующие локальные изменения как нечто, из-за чего можно просто остановить работу. Если они не мешают задаче, нужно продолжать. Если мешают, нужно чётко объяснить конфликт.

## Правило по существующим локальным изменениям

Если в репозитории уже есть незакоммиченные файлы:

- не откатывать их без прямого запроса пользователя;
- не включать их в текущий коммит автоматически, если они не относятся к задаче;
- не путать это с деплоем текущей задачи;
- всё равно завершать текущую задачу до конца: исходники + push + `gh-pages`, если правится сайт.

## Формат ответа пользователю после выполнения

После завершения задачи агент должен явно сообщить:

1. Хэш коммита в `gh-pages-source`.
2. Хэш коммита в `gh-pages`.
3. Что именно уже запушено в GitHub.
4. Что GitHub Pages обновлён.

## Приоритет

Если в старом чате агент работал по этому workflow, новый агент обязан продолжить работать точно так же, без упрощений и без пропуска этапов деплоя.
