# RUS

Обязательное правило для всех следующих сессий и агентов в этом проекте:

1. Все `git`-действия выполнять строго последовательно: `add -> commit -> push`.
2. Не запускать параллельно несколько `git`-команд в одном репозитории.
3. Перед `commit` и `push` проверять отсутствие lock-файлов:
   - `.git/index.lock`
   - `.git/worktrees/*/index.lock`
4. Если lock-файл найден и активного `git`-процесса нет, удалить lock и продолжить.
5. Для деплоя GitHub Pages:
   - сначала `gh-pages-source`,
   - потом отдельный последовательный деплой в `gh-pages`,
   - после деплоя обязательно удалять временный worktree.

Цель: исключить ошибки блокировок `index.lock` и гонки при деплое.
